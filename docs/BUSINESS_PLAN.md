## 1. Executive Summary

### 한 줄 요약

**"코딩 없이 누구나 쓸 수 있는 암호화폐 자동매매 플랫폼"**

### 사업 개요

| 항목 | 내용 |
|------|------|
| 제품명 | Upbit-Quant (가칭) |
| 형태 | SaaS (Software as a Service) |
| 대상 거래소 | 업비트 (국내 1위, 점유율 ~80%) |
| 핵심 가치 | 검증된 퀀트 전략을 비전문가도 클릭 몇 번으로 운용 |
| 운영 형태 | 1인 부업 (본업 병행, 퇴근 후/주말 개발) |
| 수익 모델 | 월 구독료 (프리미엄 티어) + 수익 수수료 (성과 보수) |

### 해결하는 문제

1. **감정적 매매**: 개인 투자자의 85%가 감정에 의한 매매로 손실 경험
2. **기술 장벽**: 자동매매를 원하지만 코딩 능력이 없는 투자자 다수
3. **시간 부족**: 24시간 시장을 사람이 지켜볼 수 없음
4. **전략 부재**: 검증된 퀀트 전략에 대한 접근성 부족

## 2. 시장 분석

### 2.1 시장 규모

| 지표 | 수치 | 출처/기준 |
|------|------|-----------|
| 국내 암호화폐 투자자 수 | 약 600만~800만 명 | 2025년 추정 |
| 업비트 일 거래량 | 3조~10조원 | 시장 상황에 따라 변동 |
| 글로벌 알고리즘 트레이딩 시장 | $21.7B (2025) → $37.6B (2030) | CAGR 11.6% |
| 국내 암호화폐 자동매매 도구 사용자 | 약 10만~30만 명 추정 | - |

### 2.2 타겟 고객 세그먼트

| 세그먼트 | 규모(추정) | 특징 | 월 지불 의향 |
|----------|------------|------|-------------|
| **1차: 코린이** | 300만+ | 암호화폐에 관심은 있지만 매매 경험 부족, 자동화 욕구 높음 | 1~3만원 |
| **2차: 경험 투자자** | 100만+ | 직접 매매 중이나 자동화/백테스팅 필요, 전략 검증 욕구 | 3~10만원 |
| **3차: 전문 트레이더** | 10만+ | 고급 전략 커스터마이징, API 접근, 대규모 자산 운용 | 10~30만원 |

### 2.3 시장 트렌드

- **자동매매 수요 증가**: MZ세대 중심으로 "자동으로 돈 벌기" 관심 급증
- **퀀트 대중화**: 유튜브/블로그를 통한 퀀트 전략 교육 콘텐츠 폭발적 증가
- **거래소 API 개방**: 업비트, 빗썸 등 국내 거래소 API 제공 확대
- **규제 안정화**: 가상자산이용자보호법 시행으로 시장 신뢰도 상승

## 3. 제품 정의

### 3.1 제품 형태

현재 CLI 기반 Python 스크립트를 **웹 기반 SaaS 플랫폼**으로 전환합니다.

```
[현재]                        [목표]
Python 스크립트              →  웹 대시보드
로컬 PC에서 실행             →  클라우드 서버 24시간 운영
.env 파일로 설정             →  웹 UI에서 클릭으로 설정
터미널 로그 확인             →  실시간 대시보드 + 모바일 알림
개발자만 사용 가능           →  누구나 회원가입 후 바로 사용
```

### 3.2 핵심 기능 (MVP → 풀 버전)

#### MVP (최소 기능 제품) - 출시 3개월 내

| 기능 | 설명 |
|------|------|
| 회원가입/로그인 | 이메일 기반 인증 |
| 업비트 API 키 등록 | 사용자가 자신의 API 키를 안전하게 등록 |
| 전략 선택 | 변동성 돌파 전략 1종 제공 |
| 원클릭 자동매매 시작/중지 | 웹에서 버튼 클릭으로 봇 ON/OFF |
| 매매 내역 조회 | 실시간 매매 기록 테이블 |
| 텔레그램 알림 연동 | 기존 텔레그램 봇 기능 유지 |
| 시뮬레이션 모드 | 실제 돈 없이 전략 테스트 |

#### 풀 버전 - 출시 6~12개월

| 기능 | 설명 |
|------|------|
| 다중 전략 제공 | 변동성 돌파, 듀얼 모멘텀, RSI+볼린저, MA크로스 |
| 백테스팅 엔진 | 과거 데이터로 전략 성능 검증 (수익률, MDD, 승률 등) |
| 수익률 대시보드 | 일별/월별 수익률 차트, 자산 추이 그래프 |
| 일일 리포트 | 매일 전략 성능 요약을 텔레그램/이메일로 전송 |
| 전략 파라미터 튜닝 | K값, RSI 기간 등을 슬라이더로 조절 |
| 포트폴리오 관리 | 코인별 비중 분배, 리밸런싱 |
| 모바일 앱 | iOS/Android 알림 및 간편 모니터링 |

### 3.3 사용자 여정 (User Journey)

```
1. 랜딩페이지 방문 → 서비스 소개 확인
2. 회원가입 (이메일/카카오/구글)
3. 무료 체험 시작 (시뮬레이션 모드 14일)
4. 업비트 API 키 등록
5. 전략 선택 및 파라미터 설정
6. "자동매매 시작" 버튼 클릭
7. 텔레그램으로 실시간 매매 알림 수신
8. 대시보드에서 수익률 확인
9. 만족 시 유료 전환
```

## 7. 기술 개발 로드맵

### 7.1 단계별 계획 (1인 부업 기준, 퇴근 후/주말 개발)

```
[Phase 1] MVP 개발 (1~4개월, 퇴근 후 개발)
    ├─ 백테스팅 엔진 완성 (현재 코어 엔진 기반)
    ├─ 웹 프론트엔드 (Next.js + Vercel)
    ├─ 사용자 인증 (Supabase Auth)
    ├─ 변동성 돌파 전략 1종 탑재
    ├─ 시뮬레이션 모드 웹 제공
    └─ 베타 테스터 10~20명 모집 (지인, 커뮤니티)

[Phase 2] 유료 전환 (5~8개월)
    ├─ 추가 전략 2~3종 구현
    ├─ 결제 시스템 연동 (토스페이먼츠)
    ├─ 수익률 대시보드 기본 구현
    ├─ 일일 리포트 텔레그램 발송
    └─ 정식 출시 + 유료 티어 오픈

[Phase 3] 안정화 (9~18개월)
    ├─ 백테스팅 결과 시각화 고도화
    ├─ 전략 파라미터 튜닝 UI
    ├─ 다중 거래소 지원 (빗썸 추가)
    └─ 매출 안정화 후 전업 전환 검토

[Phase 4] 확장 (18개월+, 전업 전환 후)
    ├─ 모바일 앱 (React Native)
    ├─ 전략 마켓플레이스
    ├─ AI 기반 시장 분석
    └─ 팀 빌딩 시작
```

> **핵심**: 스타트업처럼 빠르게 달리지 않습니다.
> 본업을 유지하며 리스크 없이 천천히, 그러나 꾸준히 만들어갑니다.

### 7.2 기술 스택 전환 계획

| 분류 | 현재 (v1.0) | 상용화 버전 |
|------|-------------|-------------|
| 프론트엔드 | 없음 (CLI) | React / Next.js |
| 백엔드 | Python 스크립트 | FastAPI (Python) |
| 데이터베이스 | Supabase | Supabase (유지) + Redis (캐싱) |
| 인프라 | 로컬 PC | AWS / GCP 클라우드 |

## 부록: 핵심 질문과 답변

### Q1. 왜 업비트만 지원하나요?

국내 암호화폐 거래량의 약 80%가 업비트에 집중되어 있습니다. 하나의 거래소에 집중하여 최적의 사용자 경험을 만든 후, Phase 3에서 빗썸/코인원/바이낸스로 확장합니다.

### Q2. 자동매매가 정말 수익을 내나요?

모든 전략에는 손실 가능성이 있습니다. 변동성 돌파 전략은 래리 윌리엄스가 1987년 세계 트레이딩 챔피언십에서 11,376% 수익률을 기록한 전략에서 파생되었지만, 과거 성과가 미래를 보장하지 않습니다. 이것이 백테스팅과 시뮬레이션 모드를 기본 제공하는 이유입니다.

### Q3. 사용자 돈을 대신 관리하나요?

아닙니다. 사용자의 업비트 계좌에서 직접 매매가 실행되며, 우리는 API 키의 **매수/매도 권한만** 사용합니다. 출금 권한은 요구하지 않으므로 자산 인출은 불가능합니다.

### Q4. 경쟁사 대비 이길 수 있는 포인트는?

"업비트 특화 + 완전 한국어 + 낮은 가격 + 투명한 전략 공개"의 조합입니다. 기존 서비스들은 글로벌 거래소 중심이거나, 영어 인터페이스이거나, 전략을 블랙박스로 제공합니다. 우리는 한국 투자자를 위해 처음부터 설계된 서비스입니다.
