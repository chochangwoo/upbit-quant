# 지금부터 할 일 - 단계별 액션 플랜

> 현재 상태: 트레이딩 엔진 (변동성 돌파) 개발 완료, 시뮬레이션 모드 동작 확인
> 목표: 1인 부업으로 SaaS 상용화
> 원칙: 본업 유지하면서, 퇴근 후/주말에 한 단계씩

---

## STEP 0. 지금 당장 해야 할 것 (이번 주)

> 돈 한 푼 안 들고, 코딩도 필요 없는 것부터

- [ ] **사업자 등록 결정하기**
  - 개인사업자 등록 (관할 세무서 또는 홈택스 온라인)
  - 업종: "소프트웨어 개발 및 공급업" 또는 "응용소프트웨어 개발업"
  - 비용: 무료 (홈택스에서 온라인 신청 가능)
  - 아직 매출이 없으므로 **간이과세자**로 시작

- [ ] **서비스 이름 확정하기**
  - "Upbit-Quant"는 가칭 → 상표권 문제 없는 이름 정하기
  - "업비트"는 두나무 상표이므로 서비스명에 직접 포함하면 안 됨
  - 예시: "CoinPilot", "퀀트메이트", "오토코인" 등 독자적 이름 필요

- [ ] **도메인 확보하기**
  - 서비스명 확정 후 .com / .kr 도메인 선점 (연 1~2만원)
  - 가비아, 카페24 등에서 구매

---

## STEP 1. 현재 제품 완성하기 (1~4주)

> 지금 있는 코드를 먼저 제대로 완성한다

### 1-1. 실거래 전환 테스트
- [ ] 업비트에서 소액(1만원) 입금
- [ ] `.env`에서 `LIVE_TRADING=true`로 변경
- [ ] 실제 매수/매도가 정상 작동하는지 확인
- [ ] Supabase `trades` 테이블에 기록이 쌓이는지 확인
- [ ] 텔레그램 알림이 오는지 확인
- [ ] 확인 후 다시 `LIVE_TRADING=false`로 복귀

### 1-2. 백테스팅 엔진 구현
- [ ] `backtest/engine.py` - 과거 데이터 로드 + 시뮬레이션 실행
- [ ] `backtest/strategies/volatility_breakout.py` - 변동성 돌파 백테스팅
- [ ] `backtest/report.py` - 수익률, MDD, 승률 계산 및 차트 출력
- [ ] 나머지 3종 전략은 나중에 (MVP에는 1종이면 충분)

### 1-3. 일일 리포트 봇 구현
- [ ] `notify/daily_report.py` - 어제 매매 요약 + 누적 수익률
- [ ] 매일 아침 텔레그램으로 자동 전송

> **왜 이걸 먼저?**: 웹 없이도 "백테스팅 결과"와 "일일 리포트"를
> 블로그/커뮤니티에 공유할 수 있다 → 마케팅 소재가 된다

---

## STEP 2. 마케팅 소재 만들기 (2~4주, STEP 1과 병행)

> 제품을 만들면서 동시에 콘텐츠를 쌓아간다

### 2-1. 블로그 개설 및 글 작성
- [ ] 티스토리 또는 브런치 블로그 개설
- [ ] 글 1: "암호화폐 자동매매란? 비전문가를 위한 쉬운 설명"
- [ ] 글 2: "변동성 돌파 전략, 비트코인 1년 백테스팅 결과 공개"
- [ ] 글 3: "내가 자동매매 봇을 직접 만든 이유"
- [ ] 글 4: "K값을 바꾸면 수익이 어떻게 달라질까? 실험 결과"
- [ ] 주 1~2편 꾸준히 발행 (SEO 누적 효과)

### 2-2. 커뮤니티 활동 시작
- [ ] 카카오 오픈채팅방 개설 ("암호화폐 자동매매 연구")
- [ ] 코인판, 클리앙, 디시 코인갤에 자연스러운 후기 공유
- [ ] 초기 관심자 10~20명 모으기 → 베타 테스터 후보

---

## STEP 3. 웹 서비스 MVP 만들기 (1~2개월)

> 핵심 기능만 넣은 최소 버전 웹사이트

### 3-1. 기술 세팅
- [ ] Next.js 프로젝트 생성
- [ ] Vercel에 연결 (무료 배포)
- [ ] Supabase Auth 연동 (회원가입/로그인)

### 3-2. MVP 필수 페이지 (4개면 충분)

| 페이지 | 기능 |
|--------|------|
| **랜딩 페이지** | 서비스 소개, 회원가입 버튼 |
| **대시보드** | 현재 봇 상태, 오늘 매매 내역 |
| **설정** | API 키 등록, 전략 선택, 코인 선택 |
| **백테스팅** | 전략 선택 → 결과 차트 표시 |

### 3-3. 핵심 백엔드
- [ ] FastAPI 서버 또는 Next.js API Routes로 구현
- [ ] 사용자별 봇 인스턴스 관리 (클라우드 서버에서 실행)
- [ ] API 키 암호화 저장 (AES-256)

---

## STEP 4. 베타 테스트 (2~3주)

- [ ] 블로그/오픈채팅에서 모은 관심자에게 베타 테스트 초대
- [ ] 10~20명 무료 사용
- [ ] 피드백 수집 (구글 폼)
- [ ] 치명적 버그 수정
- [ ] "이 서비스에 월 2만원 낼 의향이 있나요?" 설문

---

## STEP 5. 유료 전환 (1~2주)

- [ ] 토스페이먼츠 또는 포트원 연동 (정기 결제)
- [ ] Free / Basic 2개 요금제 적용
- [ ] 이용약관 + 개인정보처리방침 작성
- [ ] 투자 위험 고지문 작성 및 게시
- [ ] 통신판매업 신고 (사업자 등록 후)

---

## STEP 6. 성장 (계속)

- [ ] 블로그 글 꾸준히 발행 (주 1편 이상)
- [ ] 사용자 피드백 기반 기능 개선
- [ ] 추가 전략 구현 (듀얼 모멘텀, RSI+볼린저, MA크로스)
- [ ] 레퍼럴 프로그램 도입
- [ ] 월매출 50만원 이상 시 소규모 광고 시작

---

## 우선순위 요약

```
지금 바로     →  서비스명 확정 + 도메인 구매 + 사업자 등록
1~4주        →  실거래 테스트 + 백테스팅 엔진 + 블로그 시작
1~2개월      →  웹 MVP 개발
2~3개월      →  베타 테스트 + 피드백 반영
3~4개월      →  유료 전환 + 정식 출시
4개월~       →  콘텐츠 마케팅 + 기능 확장 + 성장
```
