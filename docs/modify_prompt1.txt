# 크립토 멀티에셋 퀀트 전략 — 롤링 윈도우 백테스트 프롬프트
# Claude Code에 아래 내용을 지시하세요.

---

## 프로젝트 개요

업비트 KRW 마켓 실데이터를 사용하여 3가지 크립토 멀티에셋 퀀트 전략을 롤링 윈도우(Walk-Forward) 방식으로 백테스트하는 파이썬 프로젝트를 구축해줘.

## 배경

- 현재 업비트에서 BTC 5/20 이동평균 크로스 전략으로 자동매매 중 (자본 약 200만 원)
- 이 전략의 한계(횡보장 whipsaw, 단일 자산 집중 리스크)를 보완하기 위해 멀티에셋 퀀트 전략을 검증하고자 함
- 특정 기간에 대한 편향을 보정하기 위해 롤링 윈도우 방식 필수

## 기술 스택 및 데이터

### 데이터 소스: 업비트 API (pyupbit 라이브러리)
- `pip install pyupbit`
- `pyupbit.get_ohlcv("KRW-BTC", interval="day", count=200)` — 한 번에 200일까지 조회
- 분당 600회, 초당 10회 제한 → 200개 초과 시 페이지네이션 필요 (period=0.5 이상 권장)
- API 키 불필요 (시세 조회는 Public API)

### 대상 코인 (업비트 KRW 마켓 확인 완료)
```python
COINS = [
    "KRW-BTC", "KRW-ETH", "KRW-SOL", "KRW-XRP", "KRW-DOGE",
    "KRW-ADA", "KRW-AVAX", "KRW-LINK", "KRW-DOT", "KRW-XLM",
    "KRW-NEAR", "KRW-UNI", "KRW-POL",  # 구 MATIC
]
```
- BNB, LTC는 업비트 KRW 마켓 미지원이므로 제외
- 13개 코인으로 진행

### 데이터 수집 기간
- 최대한 길게 (2년 이상 권장, 최소 1.5년)
- pyupbit의 `to` 파라미터로 과거 날짜를 지정하며 페이지네이션
- 수집 후 CSV로 저장하여 재사용 (`data/prices.csv`, `data/volumes.csv`)

## 백테스트할 3가지 전략

### 전략 1: 크로스섹셔널 모멘텀 (Cross-Sectional Momentum)
- 최근 N일(lookback) 수익률 기준 상위 K개 코인에 동일비중 투자
- 하위 코인은 보유하지 않음 (롱 온리)
- 거래량 필터: 최근 7일 평균 거래량 하위 20% 코인 제외
- 파라미터: lookback=[7, 14, 21], top_k=[3, 5]

### 전략 2: 변동성 가중 포트폴리오 (Risk Parity Lite)
- 전체 유효 코인에 대해 최근 N일 일별 수익률 표준편차의 역수에 비례하여 비중 배분
- 변동성 높은 코인은 적게, 낮은 코인은 많이
- 파라미터: vol_lookback=[20, 60]

### 전략 3: 통합 전략 (모멘텀 + 변동성 가중 + 거래량 시그널)
- Step 1: 모멘텀 스코어 계산 (최근 N일 수익률)
- Step 2: 거래량 변화 시그널 (최근 7일 평균 거래량 / 이전 30일 평균 거래량 - 1)
- Step 3: 복합 스코어 = 모멘텀 순위 × (1 + 거래량변화 × 0.3)
- Step 4: 상위 K개 코인을 역변동성 가중으로 비중 결정
- 파라미터: mom_lookback=[7, 14], vol_lookback=[20], top_k=[5, 7]

### 벤치마크
- BTC 바이앤홀드
- 동일비중 바이앤홀드 (13개 코인 균등)

## 백테스트 엔진 요구사항

### 롤링 윈도우 (Walk-Forward)
- 학습(In-Sample) 윈도우: 90일
- 테스트(Out-of-Sample) 윈도우: 30일
- 슬라이딩 간격: 30일 (겹치지 않는 OOS)
- 테스트 윈도우 내 리밸런싱: 7일마다

### 거래비용
- 업비트 KRW 마켓 기준: 편도 0.05%
- 리밸런싱 시 턴오버(비중 변화 절대값 합) × 0.05% 차감

### 리밸런싱 로직
- 리밸런싱 시점에 학습 윈도우를 확장(extending)하여 비중 재계산
- 리밸런싱이 아닌 날은 가격 변동에 따른 비중 드리프트만 반영

### 성과 지표 (각 전략 + 벤치마크)
- 누적 수익률, 연환산 수익률
- 연환산 변동성
- 샤프 비율 (무위험수익률 0% 가정)
- 소르티노 비율
- 최대 낙폭 (MDD)
- 칼마 비율 (연수익률 / |MDD|)
- 일별 승률
- 윈도우별 승률 (OOS 30일 구간 수익 > 0인 비율)

### 기간 편향 분석 (핵심)
- 윈도우별 수익률 분포: 평균, 중앙값, 표준편차, Q1/Q3, 최고/최저
- 시장 레짐별 성과 분류:
  - BTC 20일 MA 기울기 > 0이면 불장, < 0이면 하락장, 절대값 작으면 횡보
  - 또는 BTC 30일 수익률 기준: >+10% 불장, <-10% 하락장, 나머지 횡보
- 레짐별 윈도우 수, 평균 수익률, 승률

## 출력 요구사항

### 1. 콘솔 출력
- 데이터 수집 진행률
- 전략별/파라미터별 종합 성과 비교 테이블
- 윈도우별 수익률 분포 통계
- 레짐별 성과 요약
- 최적 파라미터 조합 추천

### 2. 파일 출력
- `results/backtest_results.json` — 전체 결과 (차트 데이터 포함)
- `results/equity_curves.csv` — 전략별 일별 누적 수익률
- `results/window_detail.csv` — 윈도우별 상세 (기간, 수익률, 샤프, MDD, 레짐, 상위 보유 종목)
- `results/summary.csv` — 전략 × 파라미터별 종합 성과

### 3. 시각화 (matplotlib 또는 plotly)
- 전략별 누적 수익률 곡선 (벤치마크 포함)
- 윈도우별 수익률 바 차트 (레짐별 색상 구분)
- 전략 간 성과 지표 레이더 차트 또는 히트맵
- 레짐별 성과 비교 그룹 바 차트

## 실행 흐름

1. `data_collector.py` — 업비트 API로 13개 코인 과거 데이터 수집 → CSV 저장
   - 이미 CSV가 있으면 스킵하고 불러오기 (캐싱)
   - 없으면 수집 후 저장
2. `main.py` — 전략 × 파라미터 조합별 백테스트 실행
3. 결과 저장 + 시각화 출력

## 추가 주의사항

- pyupbit의 get_ohlcv는 200개 초과 시 내부적으로 여러 번 요청함. period 파라미터로 요청 간격 조절 필요 (0.5초 이상)
- 코인별 상장 시점이 다르므로, 전체 코인의 공통 기간(intersection)을 사용하거나, 데이터가 있는 코인만 동적으로 포함하는 로직 필요
- 업비트 가격은 KRW 단위이므로 소수점 처리 주의 (XRP 같은 저가 코인은 원 단위 변동)
- 결과 해석 시 합성 데이터가 아닌 실데이터이므로 2024-2025 시장 상황(BTC ETF 승인 랠리, 알트시즌 등)이 반영됨을 명시
