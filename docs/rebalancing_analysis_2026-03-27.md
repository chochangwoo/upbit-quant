## 1. 현재 전략 구조

### 적응형 거래량돌파 전략 (adaptive_volume)

BTC 가격 기반으로 시장 국면을 3가지로 분류하고, 국면별로 다른 행동을 취한다.

**국면 감지 기준 (BTC 기반)**

| 국면 | 조건 | 행동 |
|------|------|------|
| Bull (상승) | BTC > SMA50 AND 20일 모멘텀 > +10% | 거래량 돌파 상위 5개 코인 매수 |
| Sideways (횡보) | 위 두 조건 어디에도 해당 안 됨 | 거래량 돌파 상위 5개 코인 매수 |
| Bear (하락) | BTC < SMA50 AND 20일 모멘텀 < -10% | 전량 매도, 현금 보유 |

**코인 선택 로직**

1. 13개 대상 코인의 최근 4일 평균 거래대금 / 20일 평균 거래대금 = vol_ratio 계산
2. vol_ratio >= 1.26 AND 4일 가격 모멘텀 > 0인 코인만 후보
3. vol_ratio * (1 + 모멘텀) 스코어 기준 상위 5개 선택
4. 5개 코인에 균등 비중 투자

**리밸런싱 주기**: 3일 (최소 대기 기간)

**BTC 국면 분포 (830일 기준)**

| 국면 | 일수 | 비중 |
|------|------|------|
| Bull | 152일 | 18.3% |
| Sideways | 582일 | 70.1% |
| Bear | 96일 | 11.6% |

## 2. 분석 1: 리밸런싱 방식 3종 비교

### 문제 인식

현재 코드는 기계적 3일 주기가 아니라, 3일 대기 후 거래량 돌파 신호가 있을 때만 리밸런싱을 실행한다.
신호가 없으면 리밸런싱이 무기한 스킵되는 구조.

### 비교 대상

| 방식 | 설명 |
|------|------|
| A. 현재방식 (신호필수) | 3일 대기 + 거래량 돌파 신호 필수. 없으면 스킵 |
| B. 기계적3일 (강제) | 3일마다 무조건 리밸런싱. 신호 없으면 모멘텀 상위로 대체 |
| C. 적응형3~7일 | 신호 있으면 3일, 없으면 최대 7일 대기 후 모멘텀으로 강제 |

## 4. 최종 결론 및 권장사항

### 권장 전략: 상승장에서만 매수, 횡보/하락장은 전량 현금

| 국면 | 행동 |
|------|------|
| Bull (상승) | 거래량 돌파 상위 5개 코인 매수 |
| Sideways (횡보) | 전량 현금 보유 |
| Bear (하락) | 전량 현금 보유 |

### 기대 효과 (830일 백테스트 기준)

| 지표 | 현재 방식 | 권장 방식 | 개선 |
|------|-----------|-----------|------|
| 수익률 | -74.6% | +41.3% | +115.9%p |
| MDD | -82.5% | -45.0% | +37.5%p |
| 샤프 | -0.80 | 0.53 | +1.33 |
| 거래수 | 1,148건 | 304건 | -73.5% |

### 트레이드오프

- 상승장 수익률이 +293% → +168%로 감소 (상승장 초입에 횡보로 분류될 수 있어 진입이 늦어짐)
- 전체 기간의 70%를 현금으로 보유하므로 기회비용 존재
- 국면 판단의 정확도에 크게 의존 (SMA50 + 20일 모멘텀 기반)

### 향후 검토사항

1. 국면 전환 시 즉시 반응 vs 확인 후 반응 (위양성 방지)
2. 횡보→상승 전환 시 매수 진입 타이밍 최적화
3. 상승장 내에서의 리밸런싱 주기 최적화 (3일 vs 5일 vs 7일)
4. 대상 코인 수(top_k) 조정 (5개 vs 3개 vs 7개)
