## 2. 제품 상세 스펙

### 2.1 사용자 흐름 (전체)

```
[신규 사용자]
    │
    ├─ 1) 랜딩페이지 방문 → 서비스 소개 확인
    │
    ├─ 2) 회원가입 (카카오 로그인 / 이메일)
    │
    ├─ 3) 온보딩 가이드 (3단계)
    │       ├─ "업비트 API 키 발급 방법" 안내
    │       ├─ API 키 입력 (출금 권한 OFF 필수 안내)
    │       └─ 텔레그램 봇 연동 (선택)
    │
    ├─ 4) 무료 체험 시작 (14일 Basic 무료)
    │       ├─ 전략 선택 (변동성 돌파)
    │       ├─ 코인 선택 (BTC, ETH 등)
    │       ├─ 금액 설정 (1회 매수 금액)
    │       └─ 시뮬레이션 모드 자동 시작
    │
    ├─ 5) 대시보드에서 결과 확인
    │       ├─ 오늘 매매 내역
    │       ├─ 누적 수익률 차트
    │       └─ 텔레그램 알림 수신
    │
    ├─ 6) 백테스팅 사용
    │       ├─ 전략 + 코인 + 기간 선택
    │       ├─ "백테스트 실행" 클릭
    │       └─ 수익률, MDD, 승률 결과 확인
    │
    ├─ 7) 실거래 전환 (유료 회원만)
    │       ├─ "실거래 모드 ON" 토글
    │       ├─ 투자 위험 동의 팝업
    │       └─ 실제 매수/매도 시작
    │
    └─ 8) 유료 결제
            ├─ 14일 체험 만료 전 결제 안내
            └─ Basic 월 19,900원 결제
```

## 3. 기술 아키텍처

### 3.1 전체 구조

```
[사용자 브라우저]
       │
       ▼
[Vercel - Next.js 프론트엔드]
       │
       ▼
[Next.js API Routes 또는 FastAPI 백엔드]
       │
       ├──▶ [Supabase]
       │       ├─ Auth (회원 인증)
       │       ├─ Database (매매내역, 설정, 백테스팅 결과)
       │       └─ Row Level Security (사용자별 데이터 격리)
       │
       ├──▶ [트레이딩 봇 프로세스]
       │       ├─ 사용자별 봇 인스턴스 관리
       │       ├─ 업비트 API 호출 (매수/매도)
       │       └─ 클라우드 서버에서 24시간 실행
       │
       ├──▶ [업비트 API]
       │       ├─ 시세 조회
       │       ├─ 잔고 조회
       │       └─ 주문 실행
       │
       └──▶ [텔레그램 Bot API]
               └─ 매매 알림 전송
```

### 3.2 기술 스택 결정

| 분류 | 선택 | 이유 |
|------|------|------|
| **프론트엔드** | Next.js 14 (App Router) | React 기반, SSR 지원, Vercel 최적화 |
| **배포** | Vercel (무료 → Pro) | Next.js 공식 호스팅, 자동 배포, 무료 SSL |
| **DB** | Supabase (기존 유지) | 이미 연동 완료, Auth 내장, 무료 티어 충분 |
| **인증** | Supabase Auth | 카카오/구글 OAuth, 이메일 로그인 기본 제공 |
| **결제** | 토스페이먼츠 | 국내 1위, 정기결제 지원, 개인사업자 가입 가능 |
| **봇 서버** | AWS Lightsail ($5/월) | 저렴, 24시간 봇 실행용 |
| **차트** | Recharts 또는 Chart.js | React 호환, 무료, 가벼움 |
| **스타일** | Tailwind CSS | 빠른 UI 개발, 디자인 시스템 불필요 |
| **UI 컴포넌트** | shadcn/ui | 무료, 고품질, Tailwind 기반 |

### 3.3 Supabase 테이블 설계

#### users (Supabase Auth 기본 제공)
```
supabase.auth.users (자동 생성)
├─ id (UUID)
├─ email
├─ created_at
└─ ...
```

#### user_settings (사용자 설정)
```sql
CREATE TABLE user_settings (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    upbit_access_key TEXT,           -- AES-256 암호화 저장
    upbit_secret_key TEXT,           -- AES-256 암호화 저장
    telegram_token   TEXT,
    telegram_chat_id TEXT,
    strategy         TEXT DEFAULT 'volatility_breakout',
    strategy_params  JSONB DEFAULT '{"k": 0.5}',
    target_coins     TEXT[] DEFAULT '{KRW-BTC}',
    order_amount     INTEGER DEFAULT 10000,
    is_live          BOOLEAN DEFAULT false,
    bot_active       BOOLEAN DEFAULT false,
    plan             TEXT DEFAULT 'free',    -- free / basic
    created_at       TIMESTAMPTZ DEFAULT NOW(),
    updated_at       TIMESTAMPTZ DEFAULT NOW()
);
```

#### trades (매매 내역 - 기존 확장)
```sql
CREATE TABLE trades (
    id              BIGSERIAL PRIMARY KEY,
    user_id         UUID REFERENCES auth.users(id),
    ticker          TEXT NOT NULL,
    trade_type      TEXT NOT NULL,       -- buy / sell
    price           NUMERIC NOT NULL,
    amount          NUMERIC NOT NULL,
    quantity        NUMERIC NOT NULL,
    strategy        TEXT,
    is_simulation   BOOLEAN DEFAULT true,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

#### backtest_results (백테스팅 결과)
```sql
CREATE TABLE backtest_results (
    id              BIGSERIAL PRIMARY KEY,
    user_id         UUID REFERENCES auth.users(id),
    strategy_name   TEXT NOT NULL,
    ticker          TEXT NOT NULL,
    start_date      DATE NOT NULL,
    end_date        DATE NOT NULL,
    initial_capital NUMERIC,
    total_return    NUMERIC,
    mdd             NUMERIC,
    win_rate        NUMERIC,
    sharpe_ratio    NUMERIC,
    total_trades    INTEGER,
    avg_hold_days   NUMERIC,
    params          JSONB,              -- {"k": 0.5} 등 전략 파라미터
    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

#### subscriptions (구독 관리)
```sql
CREATE TABLE subscriptions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    plan            TEXT DEFAULT 'free',
    status          TEXT DEFAULT 'active',  -- active / cancelled / expired
    started_at      TIMESTAMPTZ DEFAULT NOW(),
    expires_at      TIMESTAMPTZ,
    payment_key     TEXT,                   -- 토스페이먼츠 결제 키
    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

## 5. 수익 시나리오

### 5.1 보수적 시나리오 (최악의 경우)

